# New Project Initialization Workflow

This workflow is triggered by `/newproject` and scaffolds a production-ready web application with a strict separation of concerns, security guardrails, and automated quality checks.

### ⚠️ Warning
Run this in an **EMPTY** directory. This workflow will overwrite files if they already exist.
**Target OS**: macOS / Linux only. `npm pkg set` quoting may differ on Windows.

### Steps

1. **Initialize Vite + React (TypeScript)**
// turbo
```bash

```

// turbo
```bash
cat > PROJECT_RULES.md <<'EOF'
# Project Rules & Development Standards

## Project Context
> Fill this section in after scaffolding.

**Purpose:** [What this app does in 1-2 sentences]
**Data Model:** [Entity A], [Entity B]
**Key Patterns:** TanStack Query · Hooks/Services separation · Supabase RLS

---

## 1. Safety & Security
* No hardcoded secrets. Use .env variables.
* All API routes must include Zod validation.
* Sanitize user-generated content (XSS prevention).
* **IDOR Safety:** Always verify user_id via getRequiredUserId() in services.
* **Cookie Security:** Auth cookies must use HttpOnly, Secure, and SameSite=Strict. Never use document.cookie directly.

## 2. Code Quality & Naming
* **Dead Code:** Remove unused logic immediately.
* **DRY:** Abstract logic into hooks if repeated 3+ times.
* **TypeScript:** NO 'any' types. Use strict interfaces.
* **Naming:** Use useFeatureData for hooks and FeatureService for business logic.

## 3. Documentation
* Every new function must have a TSDoc/JSDoc comment explaining **Why**, not just What.
* Update components.yaml when creating new domain components.
* Run `.agent/scripts/generate-registry.sh` to refresh the auto-generated file index.

## 4. AI Agent Collaboration Protocol
* **Phase 1 — Planning:** Always produce an implementation_plan.md and wait for approval before modifying more than 2 files.
* **Phase 2 — Execution:** Implement the approved plan. Run lint/typecheck immediately after writing.
* **Phase 3 — Review (mandatory):** After coding, generate a walkthrough and initiate a Technical Debt & Logic Audit. NO fixes may be applied until the audit is explicitly approved.
* **Commit Protocol:** Use Conventional Commits format (feat:, fix:, chore:). Stage only relevant files. Always ask before pushing to GitHub.

Items 1–10 mirror the automated checks run by `/security-audit`. Complete before every production deploy.

## Automated Checks (verified by /security-audit)
- [ ] **1. Secrets Safety**: No .env files committed. Run: `git log --all -- '.env*'`
- [ ] **2. HTTP Headers**: `public/_headers` contains all 6 required headers (HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy, CSP).
- [ ] **3. CSP Directives**: No `unsafe-eval` in script-src. No bare `*` in connect-src or media-src. Flip from Report-Only to enforcing before go-live.

- [ ] **7. Cookie Hygiene**: No `document.cookie =` assignments in src/. Auth cookies use HttpOnly + Secure + SameSite=Strict.
- [ ] **8. XSS Surface**: Audit all `innerHTML`, `dangerouslySetInnerHTML`, `outerHTML` usages. Confirm each is sanitized or trusted-source only.
- [ ] **9. Dependency Audit**: Run `npm audit --audit-level=high` — zero high-severity CVEs.

- [ ] **12. Rate Limiting**: Critical endpoints (auth, sensitive actions) have rate limiting implemented.
- [ ] **13. Error Monitoring**: Production errors logged to a secure external service (e.g., Sentry, Logtail).
- [ ] **14. MFA**: Multi-factor authentication enabled for all admin/developer accounts.
- [ ] **15. Database Backups**: Automated daily backups configured and verified restorable.
```

```bash
# Edge Function shared CORS utility
# Why: getCorsHeaders(req) enforces a dynamic origin allowlist so no function
# ever needs to hardcode Access-Control-Allow-Origin: * to work locally.
cat > supabase/functions/_shared/cors.ts <<'EOF'
const ALLOWED_ORIGINS = [
  Deno.env.get('FRONTEND_URL') ?? 'http://localhost:5173',
];

/**
 * getCorsHeaders - Returns CORS headers scoped to the request origin.
 * Why: Rejects unknown origins entirely instead of falling back to localhost,
 * which would silently grant CORS access to any unrecognized caller.
 * Returns null for disallowed origins — callers must handle this as a 403.
 */
export function getCorsHeaders(req: Request): Record<string, string> | null {
  const origin = req.headers.get('Origin') ?? '';
  if (!ALLOWED_ORIGINS.includes(origin)) return null;
  return {
    'Access-Control-Allow-Origin': origin,
    'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
    'Access-Control-Allow-Methods': 'POST, GET, OPTIONS',
  };
}
EOF

# Edge Function shared auth utility
# Why: verifyAuth enforces JWT validation in one place so every Edge Function
# gets IDOR protection without duplicating auth logic.
cat > supabase/functions/_shared/auth.ts <<'EOF'
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

/**
 * verifyAuth - Validates the JWT from the Authorization header.
 * Why: Centralizes auth enforcement so no function accidentally ships unauthenticated.
 * Fails fast on missing env vars to prevent opaque runtime errors.
 * @throws If the token is missing, env vars are unset, or the user cannot be resolved.
 */
export async function verifyAuth(req: Request) {
  const supabaseUrl = Deno.env.get('SUPABASE_URL');
  const supabaseAnonKey = Deno.env.get('SUPABASE_ANON_KEY');
  if (!supabaseUrl || !supabaseAnonKey) {
    throw new Error('Server misconfiguration: SUPABASE_URL and SUPABASE_ANON_KEY must be set.');
  }

  const token = req.headers.get('Authorization')?.replace('Bearer ', '');
  if (!token) throw new Error('Missing authorization token');

  const supabase = createClient(
    supabaseUrl,
    supabaseAnonKey,
    { global: { headers: { Authorization: `Bearer ${token}` } } }
  );

  const { data: { user }, error } = await supabase.auth.getUser();
  if (error || !user) throw new Error('Unauthorized');
  return user;
}
EOF

# Env Validation Utility — uses Zod for type-safe, self-documenting env parsing.
# Why: Zod gives us both runtime validation and TypeScript inference from a single schema.
cat > src/utils/env.ts <<'EOF'
import { z } from 'zod';

/**
 * envSchema - Validates all required environment variables at startup.
 * Why: Fail-fast on missing config prevents cryptic runtime errors later.
 */
const envSchema = z.object({
  VITE_SUPABASE_URL: z.string().url({ message: 'VITE_SUPABASE_URL must be a valid URL' }),
  VITE_SUPABASE_ANON_KEY: z.string().min(1, { message: 'VITE_SUPABASE_ANON_KEY is required' }),
});

const parsed = envSchema.safeParse(import.meta.env);
if (!parsed.success) {
  console.error('❌ Invalid environment variables:', parsed.error.flatten().fieldErrors);
  throw new Error('Missing or invalid environment variables. Check .env file.');
}

export const env = {
  SUPABASE_URL: parsed.data.VITE_SUPABASE_URL,
  SUPABASE_ANON_KEY: parsed.data.VITE_SUPABASE_ANON_KEY,
} as const;
EOF

# Auth Utility
cat > src/utils/auth.ts <<'EOF'
import { supabase } from "@/integrations/supabase/client";

/**
 * getRequiredUserId - Ensures user is authenticated and returns their ID.
 * Use this in all services to enforce IDOR protection.
 */
export const getRequiredUserId = async () => {
  const { data: { user }, error } = await supabase.auth.getUser();
  if (error || !user) throw new Error("Unauthorized");
  return user.id;
};
EOF
```
